# ปรับปรุงระบบปิดยอดประจำวัน (Daily Reconciliations) - เงินทอน

## บริบทปัญหา

ขั้นตอนการทำงานจริงของร้าน:
1. **เช้า**: ทิ้งเงินทอนไว้ที่ร้าน ~800-1,000 บาท → นี่คือ `carry_forward_cash` (เงินทอนยกมา)
2. **ระหว่างวัน**: ขายของ, รับเงินสด + โอน
3. **เย็น** (ตอนปิดร้าน):
   - นับสต๊อกสินค้าคงเหลือ → Tab 1
   - บันทึกของเสีย/ชำรุด → Tab 1 (lost_damaged_qty)
   - บันทึกค่าใช้จ่าย → Tab 2
   - **กันเงินสดออกเป็นเงินทอนสำหรับวันถัดไป** → ❌ **ยังไม่มีในระบบ**
   - บันทึกยอดเงินสดที่เหลือจริง (หลังหักเงินทอน) → Tab 3
   - บันทึกยอดเงินโอน → Tab 3
   - ยืนยันปิดยอด → Tab 3

## สิ่งที่ต้องแก้ไข

### ปัญหาหลัก: ระบบยังไม่มีช่อง "เงินทอนยกไป" (Carry Forward to Next Day)

ปัจจุบัน `carry_forward_cash` ใน DB มีอยู่แล้ว แต่ใช้เป็น "เงินทอนยกมา" (จากเมื่อวาน)  
**ยังขาด**: ช่องให้พนักงานกรอก **"กันเงินทอนไว้เท่าไหร่"** สำหรับวันถัดไป

### Flow ใหม่ที่ถูกต้อง

```
เงินในลิ้นชัก (เก๊ะ) ตอนเย็น ≈ เงินทอนยกมา + ยอดขายเงินสด - ค่าใช้จ่ายจ่ายจากเก๊ะ

พนักงานนับเงินสดทั้งหมดในเก๊ะ → กรอก "ยอดเงินสดรวมในเก๊ะ"
พนักงานกัน "เงินทอน" ออกไว้ 800-1,000 บาท → กรอก "เงินทอนยกไปวันถัดไป"
ยอดเงินสดส่งมอบ = ยอดเงินสดรวมในเก๊ะ - เงินทอนยกไป
```

## สรุป Flow สมบูรณ์ (Tab 3 แบบใหม่)

### สูตรคำนวณ

| รายการ | สูตร |
|---|---|
| **ยอดขายที่ควรได้** | คำนวณจาก Tab 1 (จำนวนขาย × ราคา) |
| **ยอดรวมส่วนลด** | คำนวณจาก Tab 1 (discounted_qty × ราคา) |
| **ยอดของเสีย** | คำนวณจาก Tab 1 (lost_damaged_qty × ราคา ที่เป็นต้นทุน) |
| **เงินทอนยกมา** | ดึงจาก carry_forward ของรอบก่อน หรือกรอกเอง |
| **ยอดเงินสดรวมในเก๊ะ** | พนักงานนับเงินสดทั้งหมดในเก๊ะ |
| **เงินทอนยกไปวันถัดไป** | พนักงานกรอกว่าจะกันเงินทอนไว้เท่าไหร่ |
| **ยอดเงินสดส่งมอบ** | = ยอดเงินสดรวมในเก๊ะ - เงินทอนยกไป (คำนวณอัตโนมัติ) |
| **ยอดเงินโอน** | พนักงานกรอก |
| **ส่วนต่าง** | = (เงินสดส่งมอบ + เงินโอน) - (ยอดขายที่ควรได้ - ค่าใช้จ่าย) |

> [!IMPORTANT]
> **Key Insight**: "เงินทอนยกไป" ของวันนี้ จะกลายเป็น "เงินทอนยกมา" ของวันถัดไป โดยอัตโนมัติ

## Open Questions

> [!IMPORTANT]
> 1. **ค่าใช้จ่ายจ่ายจากไหน?** — ค่าใช้จ่ายระหว่างวัน (เช่น ค่าน้ำ ค่ากาแฟ) จ่ายจากเงินในเก๊ะ หรือจ่ายจากเงินส่วนตัว? ถ้าจ่ายจากเก๊ะ ส่วนต่างต้องหักค่าใช้จ่ายออกด้วย ถ้าจ่ายจากเงินส่วนตัว ไม่ต้องหัก
> 
> 2. **ยอดส่วนลด (discounted_qty)** — ตอนนี้มี column `discounted_qty` อยู่ใน `daily_stock_counts` แต่ยังไม่เห็นว่ามีช่องให้กรอกใน Modal แก้ไข ต้องการเพิ่มช่องกรอกส่วนลดใน Modal ด้วยไหม?
>
> 3. **สูตรส่วนต่าง** — อยากให้คำนวณส่วนต่างแบบไหน?
>    - **แบบ A**: `ส่วนต่าง = (เงินสดส่งมอบ + เงินโอน) - ยอดขายที่ควรได้`  
>      (เปรียบเทียบเงินที่ส่งมอบ กับ ยอดขายที่คำนวณได้)
>    - **แบบ B**: `ส่วนต่าง = (เงินสดรวมในเก๊ะ + เงินโอน) - (เงินทอนยกมา + ยอดขายที่ควรได้)`  
>      (เปรียบเทียบเงินทั้งหมดที่มี กับ เงินทั้งหมดที่ควรมี)
